What is HTML

HTML stands for HyperText Markup Language. It is the standard markup language used to create web pages. HTML is a combination of Hypertext and Markup language. Hypertext defines the link between web pages. A markup language is used to define the text document within the tag to define the structure of web pages.

This language is used to annotate (make notes for the computer) text so that a machine can understand it and manipulate text accordingly. Most markup languages (e.g. HTML) are human-readable. The language uses tags to define what manipulation has to be done on the text.

HTML consists of various elements, that are responsible for telling search engines how to display page content. For example, headings, lists, images, links, and more. 


<!DOCTYPE html>
<html>
<head>
    <title>First HTML Code
  </head>
  <body>
    <h2>Welcome To Web Dev
    <p>Hello World

  </body>
</html>

Explanation :

1. `<!DOCTYPE html>`: - This is a document type declaration that specifies the HTML version being used, which in this case is HTML5. - It informs the browser that the document is an HTML5 document.

2. `<html>`: - This is the root element of the HTML document. All other elements are contained within it. - It encapsulates the entire HTML content of the page.

3. `<head>`: - This is the head section of the HTML document. - It contains metadata about the document, such as the title, links to external stylesheets or scripts, and other data that isn't displayed directly on the page.

4. `<title>`: - This element defines the title of the HTML document, which appears in the browser's title bar or tab. - It provides a concise description of the page's content or purpose.

5. `<body>`: - This is the body section of the HTML document. - It contains the content that is displayed directly in the browser window.

6. `<h2>`: - This is a heading element that defines a level 2 heading. - It displays text with a larger font size and bold formatting, typically used for section headings or titles.

7. `Welcome To Web Dev`:+ - This is the text content of the `<h2>` element. - It represents the heading displayed on the webpage.

8. `<p>`: - This is a paragraph element. - It defines a block of text or content, typically used for longer passages of text.

9. `Hello World`: - This is the text content of the `<p>` element. - It represents a simple greeting message that will be displayed as a paragraph on the webpage.

Basic Structure of HTML Document

HTML (Hypertext Markup Language) is the standard markup language for creating web pages. A basic HTML document consists of the following structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Title of the document</title>
</head>
<body>

    <!-- Content of the web page goes here -->

</body>
</html>

Explanation: This HTML code represents the basic structure of an HTML document:

  • <!DOCTYPE html>: This declaration defines the document type and version of HTML being used.
  • <html>: The root element of the HTML document. It contains all other HTML elements.
  • <head>: Contains meta-information about the document, such as character set, viewport settings, and title.
  • <meta>: Defines metadata about the HTML document, such as character set and viewport settings.
  • <title>: Sets the title of the HTML document, which appears in the browser's title bar or tab.
  • <body>: Contains the visible content of the HTML document, such as text, images, links, and other elements.

HTML Elements

An HTML Element is a collection of start and end tags with the content inserted in between them. HTML elements are building blocks of web pages, representing different types of content such as headings, paragraphs, links, and images.


**Headings (h1 to h6)**

HTML headings are used to define headings or titles for sections of content. There are six levels of headings, from <h1> (the most important) to <h6> (the least important).

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

Explanation: In HTML, headings are defined using the <h1> to <h6> elements. Each heading element represents a different level of importance, with <h1> being the most important and <h6> being the least important. Headings are commonly used to structure the content of a web page and improve its accessibility and readability.



**Paragraph (<p>)**

The <p> element is used to define a paragraph of text. It is a block-level element and is commonly used to group and structure text content within an HTML document.


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Example HTML Document</title>
</head>
<body>

    <h1>This is a Heading</h1>
    
    <p>This is a paragraph of text. Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nullam nec urna finibus, vestibulum justo ac, mattis odio.</p>

</body>
</html>

Explanation: In HTML, the <p> element is used to define paragraphs of text. It contains the content of the paragraph, such as text, links, images, or other elements. Paragraphs are block-level elements, meaning they create a new block of content that typically starts on a new line and is separated from surrounding content by vertical margins.



**Unordered List (<ul>) and Ordered List (<ol>)**

The <ul> element is used to create an unordered list, while the <ol> element is used to create an ordered list. Both types of lists contain list items (<li>) that define individual items within the list. 


<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

<ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

Explanation: In HTML, the <ul> element is used to create an unordered list, which represents a list of items in no particular order. Each item in the list is defined by an <li> element. Similarly, the <ol> element is used to create an ordered list, which represents a list of items in a specific order. The order of the items is determined by the sequence of the <li> elements within the <ol> element.



**Anchor (<a>)**

The <a> element, commonly known as the anchor element, is used to create hyperlinks or links to other web pages, documents, or resources. The <a> element can include various attributes to specify the target destination, link text, and link behavior.

Here are five common attributes of the <a> element:

  • href: Specifies the URL of the linked resource.
  • target: Specifies where to open the linked resource (e.g., in a new window or tab).
  • title: Specifies extra information about the linked resource (usually displayed as a tooltip).
  • rel: Specifies the relationship between the current document and the linked resource.
  • download: Specifies that the target resource should be downloaded when clicked, instead of navigating to it.

Here are multiple ways of implementing the <a> element with different attributes: 


<!-- Basic hyperlink -->
<a href="https://www.example.com">Visit Example</a>

<!-- Hyperlink with title attribute -->
<a href="https://www.example.com" title="Go to Example">Visit Example</a>

<!-- Hyperlink with target attribute (opens in new tab) -->
<a href="https://www.example.com" target="_blank">Visit Example</a>

<!-- Hyperlink with download attribute -->
<a href="https://www.example.com/file.pdf" download>Download PDF</a>

<!-- Hyperlink with rel attribute -->
<a href="https://www.example.com" rel="nofollow">Visit Example</a>

Explanation: In the example above, we demonstrate various ways of implementing the <a> element with different attributes:

  • The first example shows a basic hyperlink with the href attribute specifying the URL of the linked resource.
  • The second example adds the title attribute to provide additional information about the linked resource.
  • The third example includes the target attribute with the value "_blank" to open the linked resource in a new tab or window.
  • The fourth example uses the download attribute to specify that the linked resource should be downloaded when clicked.
  • The fifth example demonstrates the use of the rel attribute to specify the relationship between the current document and the linked resource.


**Image (<img>)**

The <img> element is used to embed images into an HTML document. It is a self-closing tag and does not have a closing tag. The <img> element requires the src attribute to specify the URL of the image to be displayed.

Here are some common attributes of the <img> element:

  • src: Specifies the URL of the image.
  • alt: Specifies an alternate text for the image, which is displayed if the image cannot be loaded or is inaccessible to screen readers.
  • width: Specifies the width of the image in pixels or as a percentage of the parent container.
  • height: Specifies the height of the image in pixels or as a percentage of the parent container.
  • title: Specifies extra information about the image (usually displayed as a tooltip).
  • style: Specifies CSS styles to apply to the image.

Here's an example of using the <img> element:


<img src="example.jpg" alt="Example Image" width="200" height="150" title="Example Title" style="border: 1px solid #000;">

Explanation: In the example above, we use the <img> element to embed an image into the HTML document. We specify the src attribute to specify the URL of the image file. The alt attribute provides alternative text for the image, which is displayed if the image cannot be loaded or is inaccessible to screen readers. We also set the width and height attributes to control the size of the image, and the title attribute to provide extra information about the image. Additionally, we use the style attribute to apply a CSS border to the image.



Classes and IDs in HTML

Classes and IDs are attributes used to apply styles and target elements in HTML and CSS.

Class: The class attribute is used to apply one or more CSS classes to an element. Multiple elements can share the same class, and a single element can have multiple classes separated by spaces.

ID: The id attribute is used to uniquely identify an element in an HTML document. Each ID value must be unique within the document.

Here's an example of using classes and IDs:


<div id="container">
    <h2 class="heading">Welcome to our Website!</h2>
    <p class="paragraph">This is a paragraph with a <span id="highlighted">highlighted</span> word.</p>
</div>

Explanation: In the example above, we have an outer <div> element with the id attribute set to "container". Inside this container, we have an <h2> heading with the class attribute set to "heading". This allows us to apply styles to all headings with the class "heading". Similarly, we have a <p> paragraph with the class attribute set to "paragraph", and within this paragraph, we have a <span> element with the id attribute set to "highlighted". This ID can be used to target this specific element with CSS or JavaScript.



**Division (<div>) and Span (<span>)**

The <div> and <span> elements are used as generic container elements to group and style content within an HTML document.

The <div> element is a block-level element, meaning it typically starts on a new line and stretches the entire width of its parent container. It is commonly used to create sections or divisions within a web page.

The <span> element, on the other hand, is an inline element, meaning it does not start on a new line and only takes up as much width as its content. It is commonly used to apply styles or manipulate individual parts of a block of text.

Here's an example of using the <div> and <span> elements:


<div id="container">
    <h2>Welcome to our Website!</h2>
    <p>This is a <span style="color: blue;">blue</span> paragraph within a <div> element.</p>
    <div class="section">
        <p>This is another paragraph within a nested <div> element.</p>
    </div>
</div>

Explanation: In the example above, we use the <div> element with the id attribute set to "container" to create a container for the content of our webpage. Inside the <div>, we have an <h2> heading and a <p> paragraph. Within the paragraph, we use the <span> element with the style attribute to change the color of the word "blue" to blue. We also have a nested <div> element with the class attribute set to "section".



**Input (<input>)**

The <input> element is used to create form controls that allow users to input data. It is a self-closing tag and does not have a closing tag. The behavior and appearance of the input element can be customized using various attributes.

Here are some common attributes of the <input> element:

  • type: Specifies the type of input control (e.g., text, password, checkbox, radio, etc.).
  • name: Specifies the name of the input control, which is submitted with the form data.
  • value: Specifies the initial value of the input control.
  • placeholder: Specifies a short hint that describes the expected value of the input control (usually displayed when the input is empty).
  • readonly: Specifies that the input control is read-only and cannot be edited by the user.
  • disabled: Specifies that the input control is disabled and cannot be interacted with by the user.
  • required: Specifies that the input control is required and must be filled out before submitting the form.

Here's an example of using the <input> element:


<form action="/submit" method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" placeholder="Enter your username" required>

    <label for="password">Password:</label>
    <input type="password" id="password" name="password" placeholder="Enter your password" required>

    <label for="remember">Remember me:</label>
    <input type="checkbox" id="remember" name="remember">

    <input type="submit" value="Submit">
</form>

Explanation: In the example above, we use the <input> element to create form controls for a login form. We use the type attribute to specify the type of each input control (text for the username, password for the password, and checkbox for the "Remember me" option). We also use the name attribute to specify the name of each input control, which is submitted with the form data. The placeholder attribute provides a short hint for the expected value of each input control, and the required attribute specifies that the username and password fields are required to be filled out before submitting the form. Finally, we use a submit button created with another <input> element with type="submit" to submit the form.



**Form (<form>)**

The <form> element is used to create an HTML form for collecting user input. It can contain various form controls such as text fields, checkboxes, radio buttons, and buttons. The form data is typically sent to a server for processing using the action attribute, and the HTTP method used for submitting the form is specified by the method attribute.

Here are some common attributes of the <form> element:

  • action: Specifies the URL where the form data should be submitted.
  • method: Specifies the HTTP method used for submitting the form data (e.g., GET or POST). Default is GET.
  • target: Specifies where to display the response after submitting the form (e.g., _self, _blank, etc.).
  • enctype: Specifies how the form data should be encoded before sending it to the server (e.g., application/x-www-form-urlencoded, multipart/form-data, text/plain).
  • autocomplete: Specifies whether the browser should automatically complete the input values based on the user's previous input.
  • novalidate: Specifies that the form data should not be validated by the browser before submission.

Here's an example of using the <form> element:


<form action="/submit" method="post" enctype="multipart/form-data">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>

    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>

    <input type="submit" value="Submit">
</form>

Explanation: In the example above, we use the <form> element to create a login form. We specify the action attribute to indicate the URL where the form data should be submitted. The method attribute is set to post, indicating that the form data should be sent using the POST method. We use the enctype attribute to specify multipart/form-data encoding, which is commonly used when uploading files. Inside the form, we have input fields for the username and password, each marked as required to ensure they are filled out before submission. Finally, we have a submit button created with an <input> element with type="submit".

HTML Forms

HTML forms are essential for collecting information from users on web pages. They allow users to input data, such as text, selections, and submit buttons. Forms are created using the form element and consist of various form controls.



**Advanced Input Methods**

In addition to the basic usage of the <input> element, there are several advanced methods and attributes that can enhance the functionality and usability of forms.

Here are some advanced methods and attributes:

  • Checkbox: Allows users to select one or more options from a list of choices.
  • Radio Button: Allows users to select only one option from a list of choices.
  • Select: Provides a dropdown menu for users to select one option from a list of choices.
  • File Upload: Allows users to upload files from their local device.
  • Date Input: Provides a date picker for users to select a date from a calendar.
  • Range Input: Allows users to select a value from a range using a slider.
  • Color Input: Provides a color picker for users to select a color.
  • Hidden Input: Stores data on the client-side without displaying it to the user.
  • Button Input: Creates a button that can be used to trigger JavaScript functions or submit forms.

<!-- Checkbox -->
<input type="checkbox" id="checkbox" name="checkbox" value="check">
<label for="checkbox">Checkbox</label>

<!-- Radio Button -->
<input type="radio" id="radio1" name="radio" value="option1">
<label for="radio1">Option 1</label>
<input type="radio" id="radio2" name="radio" value="option2">
<label for="radio2">Option 2</label>

<!-- Select -->
<select id="select" name="select">
    <option value="option1">Option 1</option>
    <option value="option2">Option 2</option>
</select>

<!-- File Upload -->
<input type="file" id="file" name="file">

<!-- Date Input -->
<input type="date" id="date" name="date">

<!-- Range Input -->
<input type="range" id="range" name="range" min="0" max="100">

<!-- Color Input -->
<input type="color" id="color" name="color">

<!-- Hidden Input -->
<input type="hidden" id="hidden" name="hidden" value="hidden value">

<!-- Button Input -->
<button type="button" onclick="alert('Button clicked!')">Click Me</button>
<button type="submit">Submit</button>

Explanation: In the example above, we demonstrate various advanced input methods using the <input> element and other related elements. We show examples of checkboxes, radio buttons, dropdown menus (select), file uploads, date inputs, range inputs, color inputs, hidden inputs, and button inputs. Each input method has its own specific attributes and behaviors, allowing for versatile and interactive forms.



**Textarea (<textarea>)**

The <textarea> element is used to create a multiline text input field, allowing users to input large amounts of text.

Unlike the <input> element, which is used for single-line text input, the <textarea> element can handle multiple lines of text.

The text entered into a <textarea> element is submitted as part of a form, just like the value of an <input> element.

Here's an example of using the <textarea> element:


<label for="message">Message:</label>
<textarea id="message" name="message" rows="4" cols="50">
Enter your message here...
</textarea>

Explanation: In the example above, we use the <textarea> element to create a text input field for entering messages. We set the rows attribute to define the number of visible rows in the textarea, and the cols attribute to define the number of visible columns. Users can input multiline text within this textarea, and the content will be submitted along with other form data when the form is submitted.



**Select (<select>) and Option (<option>)**

The <select> element is used to create a dropdown list of options, allowing users to select one or more items from the list.

The <option> element is used to define individual options within a <select> element.

Here are some common attributes of the <select> element:

  • multiple: Specifies whether multiple options can be selected (multiple="multiple" or simply multiple).
  • size: Specifies the number of visible options when the dropdown list is expanded.
  • disabled: Specifies whether the dropdown list is disabled and cannot be interacted with by the user (disabled="disabled" or simply disabled).

Here's an example of using the <select> and <option> elements: 


<label for="cars">Choose a car:</label>
<select id="cars" name="cars">
    <option value="volvo">Volvo</option>
    <option value="saab">Saab</option>
    <option value="mercedes">Mercedes</option>
    <option value="audi">Audi</option>
</select>

Explanation: In the example above, we use the <select> element to create a dropdown list of car options. Inside the <select> element, we use multiple <option> elements to define individual car options. Each <option> element has a value attribute that specifies the value of the selected option, which is submitted with the form data. The text content of each <option> element represents the visible label of the option in the dropdown list.



**Button (<button>)**

The <button> element is used to create a clickable button on a webpage. It can be used to trigger JavaScript functions, submit form data, or perform other user-defined actions.

Unlike the <input> element with type="button", the <button> element can contain other elements such as text, images, or even nested HTML elements.

Here's an example of using the <button> element:


<button type="button">Click Me</button>

Explanation: In the example above, we use the <button> element to create a clickable button with the text "Click Me". The type="button" attribute explicitly specifies that this button should not submit the form when clicked. Instead, it triggers a JavaScript function or performs other actions as defined by the developer.

HTML Tables

HTML tables are used to display data in rows and columns format. They are comprised of the following main elements:



**Table (<table>)**

The <table> element is used to create tables in HTML, allowing you to organize and display tabular data in rows and columns.

A table is made up of multiple elements:

  • <table>: The container element for the entire table.
  • <tr>: Table rows, each containing one or more table data or table header cells.
  • <th>: Table header cells, used to define column headers.
  • <td>: Table data cells, used to define data within the table.
  • <caption>: Optional caption for the table.

Here's an example of using the <table> element:


<table border="1">
    <caption>Monthly Expenses</caption>
    <tr>
        <th>Category</th>
        <th>Amount</th>
    </tr>
    <tr>
        <td>Food</td>
        <td>$200</td>
    </tr>
    <tr>
        <td>Transportation</td>
        <td>$100</td>
    </tr>
    <tr>
        <td>Utilities</td>
        <td>$150</td>
    </tr>
</table>

Explanation: In the example above, we use the <table> element to create a simple table displaying monthly expenses. We use <tr> elements to define each row of the table, with <th> elements used for the table header row and <td> elements used for the data rows. The optional <caption> element provides a title for the table. We use the border attribute to specify a border around the table.



**Table Caption (<caption>)**

The <caption> element is used to add a title or caption to an HTML table. It provides a brief description or summary of the table content.

The <caption> element should be placed immediately after the opening <table> tag, but before any <tr> elements.

Here's an example of using the <caption> element:


<table border="1">
    <caption>Monthly Expenses</caption>
    <tr>
        <th>Category</th>
        <th>Amount</th>
    </tr>
    <tr>
        <td>Food</td>
        <td>$200</td>
    </tr>
    <tr>
        <td>Transportation</td>
        <td>$100</td>
    </tr>
    <tr>
        <td>Utilities</td>
        <td>$150</td>
    </tr>
</table>

Explanation: In the example above, we use the <caption> element to add a title to the table, which reads "Monthly Expenses". This provides a brief description of the table content, making it easier for users to understand the purpose of the table.



**Table Head (<thead>), Table Body (<tbody>), and Table Footer (<tfoot>)**

The <thead> element is used to group the header content in an HTML table. It typically contains one or more rows (<tr>) of header cells (<th>), defining column headings.

The <tbody> element is used to group the main content of the table. It contains one or more rows of table data cells (<td>), representing the actual data.

The <tfoot> element is used to group the footer content in an HTML table. It typically contains one or more rows of footer cells, which can be used to display summary information or totals.

Here's an example of using the <thead>, <tbody>, and <tfoot> elements: 


<table border="1">
    <caption>Sales Report</caption>
    <thead>
        <tr>
            <th>Month</th>
            <th>Revenue</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>January</td>
            <td>$1000</td>
        </tr>
        <tr>
            <td>February</td>
            <td>$1500</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="2">Total</td>
            <td>$2500</td>
        </tr>
    </tfoot>
</table>

Explanation: In the example above, we use the <thead> element to group the header content of the table, which includes column headings such as "Month" and "Revenue". Inside the <thead>, we use <tr> and <th> elements to define the rows and header cells respectively.

The main content of the table, containing actual data like month names and revenue figures, is grouped within the <tbody> element. Each row of data is represented by a <tr> element, with data cells (<td>) containing the actual data.

The <tfoot> element is used to group the footer content of the table, which in this case contains a summary row with the total revenue. We use the colspan attribute to span the summary row across both columns, indicating that it covers the entire width of the table.



**Table Row Span (rowspan) and Column Span (colspan)**

The rowspan attribute is used to specify the number of rows a table cell should span vertically, effectively merging multiple rows into a single cell.

The colspan attribute is used to specify the number of columns a table cell should span horizontally, effectively merging multiple columns into a single cell.

Here's an example of using the rowspan and colspan attributes:


<table border="1">
    <caption>Merged Cells Example</caption>
    <tr>
        <th>Month</th>
        <th colspan="2">Revenue</th>
    </tr>
    <tr>
        <td>January</td>
        <td rowspan="2">$1000</td>
        <td>Product A</td>
    </tr>
    <tr>
        <td>February</td>
        <td>Product B</td>
    </tr>
</table>

Explanation: In the example above, we have a table representing monthly revenue for two products. We use the colspan="2" attribute in the second header cell to span across two columns, indicating that it covers both "Product A" and "Product B" columns.

In the first data row, the cell containing "January" spans two rows vertically using the rowspan="2" attribute, effectively merging the two rows below it into a single cell. This cell contains the revenue value for January, and the corresponding cells for "Product A" and "Product B" are left empty.

The second data row contains data for February, and the "Revenue" cell for "Product A" is empty since it's already merged with the January revenue cell.



**Table Borders (border)**

The border attribute is used to specify whether a table should have borders or not. When the border attribute is set to a non-zero value, borders will be displayed around the table and its cells.

Here's an example of using the border attribute:


<table border="1">
    <caption>Sales Report</caption>
    <tr>
        <th>Month</th>
        <th>Revenue</th>
    </tr>
    <tr>
        <td>January</td>
        <td>$1000</td>
    </tr>
    <tr>
        <td>February</td>
        <td>$1500</td>
    </tr>
</table>

Explanation: In the example above, we use the border="1" attribute within the <table> tag to specify that the table should have a border. The value "1" indicates that a border should be displayed around the table and its cells.

The table contains a caption "Sales Report", followed by rows and columns of data representing monthly revenues. The borders help visually separate the cells and improve the readability of the table.



**Table Cell Padding (cellpadding) and Cell Spacing (cellspacing)**

HTML Semantic Elements

Semantic elements are HTML elements that carry meaning and convey information about the structure of the web page to both the browser and the developer. They describe the purpose of the content they contain, making it easier to understand the document's structure and improve accessibility and search engine optimization (SEO). Non-semantic elements, on the other hand, are HTML elements that do not inherently carry meaning or convey information about the structure of the document. They are typically used for styling or layout purposes. Examples of non-semantic elements include `<div>`, `<span>`, `<b>`, `<i>`, `<font>`, and others.

    Your browser does not support the audio element.
