What is XML?


<?xml version="1.0" encoding="UTF-8"?>
<note>
    <to>Tove</to>
    <from>Jani</from>
    <heading>Reminder</heading>
    <body>Don't forget me this weekend!</body>
</note>

XML (eXtensible Markup Language) is a markup language that defines a set of rules for encoding documents in a format that is both human-readable and machine-readable. It provides a way to structure and organize data using tags, similar to HTML.

XML is widely used in Android app development for defining layouts, data structures, and configuration files. In Android, XML files are used to describe the user interface of an app (layouts), as well as for storing data in various formats such as preferences, resources, and more.

The example above demonstrates a simple XML document representing a note. It contains elements like <to>, <from>, <heading>, and <body>, each enclosed within a pair of angle brackets. These elements define the structure and content of the note.

Basic XML Structure


<?xml version="1.0" encoding="UTF-8"?>
<root_element>
    <child_element attribute="value">Content</child_element>
</root_element>

Basic XML Structure:
In XML, documents are structured as a hierarchy of elements enclosed within angle brackets. Each element can have attributes and may contain text content or other elements.

Layouts

Layouts:
In Android, layouts are used to define the structure and arrangement of user interface components within an activity or fragment. Different layout types offer various ways to organize and display UI elements on the screen.

LinearLayout:
LinearLayout arranges its child views either horizontally or vertically in a single direction. 


<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:orientation="vertical">
    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 1"/>
    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 2"/>
</LinearLayout>

GridView:
GridView displays its child views in a grid arrangement, typically used for displaying items in a grid-like fashion, such as images or data. 


<GridView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:numColumns="2">
    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:src="@drawable/image1"/>
    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:src="@drawable/image2"/>
</GridView>

TableLayout:
TableLayout arranges its child views into rows and columns, similar to an HTML table. 


<TableLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">
    <TableRow>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 1, Column 1"/>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 1, Column 2"/>
    </TableRow>
    <TableRow>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 2, Column 1"/>
        <TextView
            android:layout_width="wrap_content"

            android:text="Row 2, Column 2"/>

</TableLayout>

RelativeLayout:
RelativeLayout arranges its child views relative to one another or to the parent, allowing for flexible UI designs. 


<RelativeLayout

FrameLayout is a simple layout used to contain a single view or fragment, often used for displaying fragments dynamically.

ConstraintLayout:
ConstraintLayout allows you to create large and complex layouts with a flat view hierarchy (no nested layouts). It allows you to create constraints between views to define their position relative to each other or to the parent layout. 


<androidx.constraintlayout.widget.ConstraintLayout
    android:layout_width="match_parent"

        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent"/>
</androidx.constraintlayout.widget.ConstraintLayout>

ImageView is a UI component in Android used to display images. It can load images from various sources such as resources, assets, or network URLs, and supports different image formats such as JPEG, PNG, GIF, etc.


<ImageView

Explanation: In this example, we define an ImageView element in an XML layout file. Here's an explanation of the attributes used:

  • android:src: Specifies the image resource to be displayed. It can be a drawable resource, asset, or network URL.
  • android:scaleType: Defines how the image should be scaled and/or cropped to fit within the ImageView's bounds. Common values include centerCrop, fitCenter, centerInside, etc.
  • android:contentDescription: Provides a textual description of the image for accessibility purposes.

VideoView in Android

VideoView is a UI component in Android that allows playing video files. It is typically used to embed videos in an Android application.

XML Layout (activity_main.xml)

<VideoView
   android:id="@+id/videoView"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"/>
Java Code (MainActivity.java)

import android.net.Uri;
import android.os.Bundle;
import android.widget.MediaController;
import android.widget.VideoView;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       VideoView videoView = findViewById(R.id.videoView);

       // Set video URL
       String videoUrl = "https://www.example.com/video.mp4";
       Uri uri = Uri.parse(videoUrl);

       // Set MediaController
       MediaController mediaController = new MediaController(this);
       mediaController.setAnchorView(videoView);
       videoView.setMediaController(mediaController);

       // Set video URI
       videoView.setVideoURI(uri);

       // Start playing video
       videoView.start();
   }
}

AudioPlayer in Android

AudioPlayer is a UI component in Android that allows playing audio files. It is typically used to embed audio playback functionality in an Android application.

XML Layout (activity_main.xml)

<LinearLayout
   xmlns:android="http://schemas.android.com/apk/res/android"
   android:layout_width="match_parent"
   android:layout_height="match_parent"
   android:orientation="vertical">

   <Button
       android:id="@+id/buttonPlay"
       android:layout_width="wrap_content"
       android:layout_height="wrap_content"
       android:text="Play"/>

   <Button
       android:id="@+id/buttonPause"
       android:layout_width="wrap_content"
       android:layout_height="wrap_content"
       android:text="Pause"/>

   <Button
       android:id="@+id/buttonStop"
       android:layout_width="wrap_content"
       android:layout_height="wrap_content"
       android:text="Stop"/>

</LinearLayout>
Java Code (MainActivity.java)

import android.media.MediaPlayer;
import android.net.Uri;
import android.os.Bundle;
import android.view.View;
import android.widget.Button;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity implements View.OnClickListener {

   private MediaPlayer mediaPlayer;

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       Button buttonPlay = findViewById(R.id.buttonPlay);
       Button buttonPause = findViewById(R.id.buttonPause);
       Button buttonStop = findViewById(R.id.buttonStop);

       buttonPlay.setOnClickListener(this);
       buttonPause.setOnClickListener(this);
       buttonStop.setOnClickListener(this);

       // Initialize MediaPlayer
       mediaPlayer = new MediaPlayer();
       mediaPlayer.setOnPreparedListener(new MediaPlayer.OnPreparedListener() {
           @Override
           public void onPrepared(MediaPlayer mp) {
               mediaPlayer.start();
           }
       });

       // Set audio URL
       String audioUrl = "https://www.example.com/audio.mp3";
       Uri uri = Uri.parse(audioUrl);

       try {
           mediaPlayer.setDataSource(MainActivity.this, uri);
           mediaPlayer.prepareAsync(); // Prepare asynchronously to avoid blocking the UI thread
       } catch (Exception e) {
           e.printStackTrace();
       }
   }

   @Override
   public void onClick(View v) {
       switch (v.getId()) {
           case R.id.buttonPlay:
               if (!mediaPlayer.isPlaying()) {
                   mediaPlayer.start();
               }
               break;
           case R.id.buttonPause:
               if (mediaPlayer.isPlaying()) {
                   mediaPlayer.pause();
               }
               break;
           case R.id.buttonStop:
               if (mediaPlayer.isPlaying()) {
                   mediaPlayer.stop();
                   mediaPlayer.prepareAsync();
               }
               break;
       }
   }

   @Override
   protected void onDestroy() {
       super.onDestroy();
       if (mediaPlayer != null) {
           mediaPlayer.release();
       }
   }
}

Explanation: In this example, we define a ScrollView element in an XML layout file. The ScrollView contains a LinearLayout as its child, which holds the scrollable content. Here's an explanation of the attributes used:

CheckBox in Android

CheckBox is a UI component in Android that allows users to make a binary choice, such as selecting or deselecting an option. It is typically used in forms or settings screens.

XML Layout (activity_main.xml)

<CheckBox
    android:id="@+id/checkBox"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Check me"
    android:checked="false"/>
Java Code (MainActivity.java)

import android.os.Bundle;
import android.widget.CheckBox;
import android.widget.CompoundButton;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        CheckBox checkBox = findViewById(R.id.checkBox);

        // Method 1: Using setOnCheckedChangeListener
        checkBox.setOnCheckedChangeListener(new CompoundButton.OnCheckedChangeListener() {
            @Override
            public void onCheckedChanged(CompoundButton buttonView, boolean isChecked) {
                if (isChecked) {
                    Toast.makeText(MainActivity.this, "CheckBox checked", Toast.LENGTH_SHORT).show();
                } else {
                    Toast.makeText(MainActivity.this, "CheckBox unchecked", Toast.LENGTH_SHORT).show();
                }
            }
        });

        // Method 2: Retrieving CheckBox state
        boolean isChecked = checkBox.isChecked();
        if (isChecked) {
            Toast.makeText(MainActivity.this, "CheckBox is checked", Toast.LENGTH_SHORT).show();
            // CheckBox is checked
        } else {
            // CheckBox is not checked
            Toast.makeText(MainActivity.this, "CheckBox is not checked", Toast.LENGTH_SHORT).show();
        }
    }
}

RadioButton in Android

RadioButton is a UI component in Android that allows users to select one option from a group of options. It is typically used in radio groups to represent mutually exclusive choices.

<RadioGroup

        android:id="@+id/radioButton2"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Option 2"/>

</RadioGroup>
Java Code (MainActivity.java)

import android.os.Bundle;
import android.widget.RadioButton;
import android.widget.RadioGroup;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        RadioGroup radioGroup = findViewById(R.id.radioGroup);
        RadioButton radioButton1 = findViewById(R.id.radioButton1);
        RadioButton radioButton2 = findViewById(R.id.radioButton2);

        // Method 1: Using setOnCheckedChangeListener
        radioGroup.setOnCheckedChangeListener(new RadioGroup.OnCheckedChangeListener() {
            @Override
            public void onCheckedChanged(RadioGroup group, int checkedId) {
                RadioButton radioButton = findViewById(checkedId);
                String selectedOption = radioButton.getText().toString();
                Toast.makeText(MainActivity.this, "Selected option: " + selectedOption, Toast.LENGTH_SHORT).show();
            }
        });

        // Method 2: Retrieving selected RadioButton
        RadioButton selectedRadioButton = findViewById(radioGroup.getCheckedRadioButtonId());
        if (selectedRadioButton != null) {
            String selectedOption = selectedRadioButton.getText().toString();
            Toast.makeText(MainActivity.this, "Selected option: " + selectedOption, Toast.LENGTH_SHORT).show();
        }
    }
}

Spinner in Android

Spinner is a UI component in Android that provides a dropdown menu with a list of selectable items. It is typically used to allow users to choose one item from a list of options.

XML Layout (activity_main.xml)

<Spinner
    android:id="@+id/spinner"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:entries="@array/countries"/>
Java Code (MainActivity.java)

import android.os.Bundle;
import android.view.View;
import android.widget.AdapterView;
import android.widget.ArrayAdapter;
import android.widget.Spinner;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Spinner spinner = findViewById(R.id.spinner);
        ArrayAdapter adapter = ArrayAdapter.createFromResource(this,
                R.array.countries, android.R.layout.simple_spinner_item);
        adapter.setDropDownViewResource(android.R.layout.simple_spinner_dropdown_item);
        spinner.setAdapter(adapter);

        // Method 1: Using setOnItemSelectedListener
        spinner.setOnItemSelectedListener(new AdapterView.OnItemSelectedListener() {
            @Override
            public void onItemSelected(AdapterView parent, View view, int position, long id) {
                String selectedCountry = parent.getItemAtPosition(position).toString();
                Toast.makeText(MainActivity.this, "Selected country: " + selectedCountry, Toast.LENGTH_SHORT).show();
            }

            @Override
            public void onNothingSelected(AdapterView parent) {
                // Do nothing
            }
        });

        // Method 2: Retrieving selected item
        String selectedCountry = spinner.getSelectedItem().toString();
        Toast.makeText(MainActivity.this, "Selected country: " + selectedCountry, Toast.LENGTH_SHORT).show();
    }
}

SeekBar with Audio Player

SeekBar is a UI component in Android that allows users to select a value from a range by sliding a thumb along a horizontal track. It is often used to control playback progress in media players.

XML Layout (activity_main.xml)

<SeekBar
   android:id="@+id/seekBar"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:max="100"
   android:progress="0"
   android:layout_marginTop="16dp"/>
Java Code (MainActivity.java)

import android.media.MediaPlayer;
import android.os.Bundle;
import android.widget.SeekBar;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

   private MediaPlayer mediaPlayer;
   private SeekBar seekBar;
   private boolean isPlaying = false;

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       mediaPlayer = MediaPlayer.create(this, R.raw.sample_audio);
       seekBar = findViewById(R.id.seekBar);

       // Set seekbar listener
       seekBar.setOnSeekBarChangeListener(new SeekBar.OnSeekBarChangeListener() {
           @Override
           public void onProgressChanged(SeekBar seekBar, int progress, boolean fromUser) {
               if (fromUser) {
                   mediaPlayer.seekTo(progress);
               }
           }

           @Override
           public void onStartTrackingTouch(SeekBar seekBar) {
               // No implementation needed
           }

           @Override
           public void onStopTrackingTouch(SeekBar seekBar) {
               // No implementation needed
           }
       });

       // Start audio playback
       mediaPlayer.start();
       isPlaying = true;

       // Update seekbar progress
       updateSeekBar();
   }

   // Method to update seekbar progress
   private void updateSeekBar() {
       if (mediaPlayer.isPlaying()) {
           int currentPosition = mediaPlayer.getCurrentPosition();
           seekBar.setProgress(currentPosition);
           Runnable runnable = new Runnable() {
               @Override
               public void run() {
                   updateSeekBar();
               }
           };
           seekBar.postDelayed(runnable, 1000); // Update every second
       }
   }

   @Override
   protected void onDestroy() {
       super.onDestroy();
       mediaPlayer.release();
       mediaPlayer = null;
   }
}

Switch is a UI component in Android that allows users to toggle between two states, such as on/off or light/dark mode. It is often used to enable or disable specific features or settings.

XML Layout (activity_main.xml)

   android:id="@+id/switchMode"
   android:layout_width="wrap_content"
   android:layout_height="wrap_content"

   android:layout_marginTop="16dp"/>
Java Code (MainActivity.java)

import android.content.res.Configuration;
import android.os.Bundle;
import android.widget.CompoundButton;
import android.widget.Switch;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       Switch switchMode = findViewById(R.id.switchMode);

       // Check the current theme mode and set the switch accordingly
       int currentNightMode = getResources().getConfiguration().uiMode & Configuration.UI_MODE_NIGHT_MASK;
       switchMode.setChecked(currentNightMode == Configuration.UI_MODE_NIGHT_YES);

       // Set switch listener to toggle between light and dark mode
       switchMode.setOnCheckedChangeListener(new CompoundButton.OnCheckedChangeListener() {
           @Override
           public void onCheckedChanged(CompoundButton buttonView, boolean isChecked) {
               if (isChecked) {
                   // Switch to dark mode
                   getDelegate().setLocalNightMode(AppCompatDelegate.MODE_NIGHT_YES);
               } else {
                   // Switch to light mode
                   getDelegate().setLocalNightMode(AppCompatDelegate.MODE_NIGHT_NO);
               }
               recreate();
           }
       });
   }
}

Explanation : The above code demonstrates the implementation of a Switch in Android to toggle between light and dark mode. Here's a breakdown of the code:

  • XML Layout: Defines a Switch component in the layout XML file (`activity_main.xml`). The Switch is initialized with a text label "Dark Mode" and starts unchecked.
  • Java Code: Initializes the Switch in the `onCreate()` method of the MainActivity. It checks the current theme mode and sets the Switch accordingly. A listener is set on the Switch to detect changes in its state. When the Switch is toggled, it updates the app's theme to either light or dark mode and recreates the activity to apply the changes.

<DatePicker

Java Code (MainActivity.java)

import android.os.Bundle;
import android.widget.DatePicker;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;
import java.util.Calendar;

public class MainActivity extends AppCompatActivity {

   @Override
   protected void onCreate(Bundle savedInstanceState) {
       super.onCreate(savedInstanceState);
       setContentView(R.layout.activity_main);

       DatePicker datePicker = findViewById(R.id.datePicker);

       // Set initial date to current date
       Calendar calendar = Calendar.getInstance();
       int year = calendar.get(Calendar.YEAR);
       int month = calendar.get(Calendar.MONTH);
       int day = calendar.get(Calendar.DAY_OF_MONTH);
       datePicker.init(year, month, day, new DatePicker.OnDateChangedListener() {
           @Override
           public void onDateChanged(DatePicker view, int year, int monthOfYear, int dayOfMonth) {
               // Display selected date
               String selectedDate = dayOfMonth + "/" + (monthOfYear + 1) + "/" + year;
               Toast.makeText(MainActivity.this, "Selected Date: " + selectedDate, Toast.LENGTH_SHORT).show();
           }
       });
   }
}
