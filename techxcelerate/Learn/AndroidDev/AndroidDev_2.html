What is XML?


<?xml version="1.0" encoding="UTF-8"?>
<note>
    <to>Tove</to>
    <from>Jani</from>
    <heading>Reminder</heading>
    <body>Don't forget me this weekend!</body>
</note>

XML (eXtensible Markup Language) is a markup language that defines a set of rules for encoding documents in a format that is both human-readable and machine-readable. It provides a way to structure and organize data using tags, similar to HTML.

XML is widely used in Android app development for defining layouts, data structures, and configuration files. In Android, XML files are used to describe the user interface of an app (layouts), as well as for storing data in various formats such as preferences, resources, and more.

The example above demonstrates a simple XML document representing a note. It contains elements like <to>, <from>, <heading>, and <body>, each enclosed within a pair of angle brackets. These elements define the structure and content of the note.

Basic XML Structure


<?xml version="1.0" encoding="UTF-8"?>
<root_element>
    <child_element attribute="value">Content</child_element>
</root_element>

Basic XML Structure:
In XML, documents are structured as a hierarchy of elements enclosed within angle brackets. Each element can have attributes and may contain text content or other elements.

Example 1: TextView
TextView is a fundamental UI component used to display text content in an Android app. 


<TextView
    android:id="@+id/textView"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello, World!"
    android:textSize="18sp"
    android:textColor="#000000"/>

Example 2: Button
Button is a UI component used to trigger actions or events when clicked by the user. 


<Button
    android:id="@+id/button"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Click Me"
    android:onClick="onButtonClick"/>

Example 3: EditText
EditText is a UI component used to capture user input, such as text or numbers. 


<EditText
    android:id="@+id/editText"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:hint="Enter your name"/>

Example 4: ImageView
ImageView is a UI component used to display images in an Android app. 


<ImageView
    android:id="@+id/imageView"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:src="@drawable/image"/>

Example 5: CheckBox
CheckBox is a UI component used to allow users to select one or more options from a set of choices. 


<CheckBox
    android:id="@+id/checkBox"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Check me"/>

Layouts

Layouts:
In Android, layouts are used to define the structure and arrangement of user interface components within an activity or fragment. Different layout types offer various ways to organize and display UI elements on the screen.

LinearLayout:
LinearLayout arranges its child views either horizontally or vertically in a single direction. 


<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:orientation="vertical">
    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 1"/>
    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 2"/>
</LinearLayout>

GridView:
GridView displays its child views in a grid arrangement, typically used for displaying items in a grid-like fashion, such as images or data. 


<GridView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:numColumns="2">
    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:src="@drawable/image1"/>
    <ImageView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:src="@drawable/image2"/>
</GridView>

TableLayout:
TableLayout arranges its child views into rows and columns, similar to an HTML table. 


<TableLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">
    <TableRow>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 1, Column 1"/>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 1, Column 2"/>
    </TableRow>
    <TableRow>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 2, Column 1"/>
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Row 2, Column 2"/>
    </TableRow>
</TableLayout>

RelativeLayout:
RelativeLayout arranges its child views relative to one another or to the parent, allowing for flexible UI designs. 


<RelativeLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">
    <Button
        android:id="@+id/button1"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 1"/>
    <Button
        android:id="@+id/button2"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 2"
        android:layout_below="@id/button1"/>
</RelativeLayout>

AbsoluteLayout:
AbsoluteLayout positions its child views at specific coordinates, which may result in UI elements overlapping on different screen sizes. 


<AbsoluteLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">
    <Button
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button 1"
        android:layout_x="50dp"
        android:layout_y="100dp"/>
</AbsoluteLayout>

ListView:
ListView displays a scrollable list of items, allowing users to select one or more items from the list. 


<ListView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"/>

FrameLayout:
FrameLayout is a simple layout used to contain a single view or fragment, often used for displaying fragments dynamically. 


<FrameLayout
  android:layout_width="match_parent"
  android:layout_height="wrap_content">
  <TextView
      android:layout_width="wrap_content"
      android:layout_height="wrap_content"
      android:text="This is a FrameLayout"/>
</FrameLayout>

ConstraintLayout:
ConstraintLayout allows you to create large and complex layouts with a flat view hierarchy (no nested layouts). It allows you to create constraints between views to define their position relative to each other or to the parent layout. 


<androidx.constraintlayout.widget.ConstraintLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Button"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent"/>
</androidx.constraintlayout.widget.ConstraintLayout>
