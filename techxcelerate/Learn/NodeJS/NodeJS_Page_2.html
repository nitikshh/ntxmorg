Introduction to Programming

Node.js is a powerful runtime environment for executing JavaScript code on the server-side. It is built on the V8 JavaScript engine, which powers Google Chrome, and provides a rich set of features for building scalable and high-performance web applications.

  • JavaScript Runtime: Node.js allows developers to run JavaScript code outside of a web browser, making it possible to build server-side applications using JavaScript.
  • Event-Driven Architecture: Node.js uses an event-driven, non-blocking I/O model, allowing applications to handle multiple concurrent operations efficiently.
  • npm (Node Package Manager): npm is the default package manager for Node.js, providing access to a vast ecosystem of open-source libraries and modules.
  • Common Use Cases: Node.js is commonly used for building web servers, RESTful APIs, real-time applications, and command-line tools.
  • Scalability: Node.js applications can be easily scaled horizontally by adding more instances to handle increased load, making it suitable for handling large volumes of traffic.
  • Community and Support: Node.js has a large and active community of developers who contribute to its development, provide support, and create useful resources such as libraries, frameworks, and tutorials.

NPM Node.js

NPM (Node Package Manager) is the default package manager for Node.js, used for installing, managing, and sharing packages of JavaScript code. NPM provides access to thousands of reusable packages, making it easy to extend the functionality of your Node.js applications. Let's explore some NPM basics in Node.js.


Installation

NPM is included with Node.js, so you don't need to install it separately. To check if NPM is installed, you can run:

npm --version

Usage

You can use NPM to install packages, manage dependencies, and run scripts:

// Install a package locally
npm install express

// Install a package globally
npm install -g nodemon

// Install packages listed in package.json
npm install

// Run scripts defined in package.json
npm run start

Explanation: NPM simplifies the process of managing dependencies and running scripts in Node.js projects. By leveraging the vast ecosystem of NPM packages, you can easily enhance the functionality of your applications and streamline development workflows.

Connecting Node.js with HTML

You can connect Node.js with HTML to serve dynamic web pages or build web applications. Follow the steps below to create a basic setup.


File Paths
  • Node.js File: The Node.js file where you define your server logic, e.g., server.js.
  • HTML File: The HTML file you want to serve, e.g., index.html.

Code Examples

// server.js
const express = require('express');
const path = require('path');
const app = express();

// Route to serve HTML file
app.get('/', (req, res) => {
   res.sendFile(path.join(__dirname, 'index.html'));
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
   console.log('Server is running on port', PORT);
});
       

<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Node.js with HTML</title>
</head>
<body>
   <h1>Hello from Node.js!</h1>
   <p>This is a dynamic web page served by Node.js.</p>
</body>
</html>

Explanation: In the Node.js file (server.js), we use Express.js to create a server. We define a route to handle the root endpoint ('/') and serve the HTML file (index.html). In the HTML file, we create a basic HTML structure with a title, heading, and paragraph to demonstrate dynamic content served by Node.js.

Variables and Data Types in Node.js

In Node.js, variables are used to store data values, and data types define the type of data that can be stored in a variable. Let's explore variables and data types in Node.js and how they relate to passing data between the server and HTML.


Variables and Data Types
  • Variables: Variables are containers for storing data values. In JavaScript, variables can be declared using the var, let, or const keywords.
  • Data Types: Data types define the type of data that can be stored in a variable. Common data types in Node.js include strings, numbers, booleans, objects, arrays, and functions.

Passing Variables and Data Types

In a Node.js server, variables and data types can be passed to HTML files to dynamically generate content. This can be achieved by embedding variables or data values in HTML templates using templating engines such as EJS (Embedded JavaScript).


Code Example

// server.js
const express = require('express');
const path = require('path');
const app = express();

// Set up view engine and views directory
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Route to serve HTML file with dynamic content
app.get('/', (req, res) => {
   // Data to pass to HTML
   const name = 'John';
   const age = 30;

   // Render HTML with data
   res.render('index', { name, age });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
   console.log('Server is running on port', PORT);
});
       


<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Node.js with HTML</title>
</head>
<body>
   <h1>Hello, <%= name %>!</h1>
   <p>You are <%= age %> years old.</p>
</body>
</html>

Explanation: In the above example, we define variables name and age in the Node.js server. We then pass these variables to the HTML file (index.html) using a templating engine. In the HTML file, we can access these variables and dynamically generate content based on their values.

Control Structures: If...Else in Node.js

In Node.js, you can use control structures like if...else to execute different code blocks based on specified conditions. Let's see an example of how to check conditions using if...else in Node.js and display the result in HTML.


Example: Checking Marks

Suppose we have a Node.js server that checks the marks of a student and displays a message based on the marks obtained.


Code Example

// server.js
const express = require('express');
const path = require('path');
const app = express();

// Set up view engine and views directory
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Route to serve HTML file with dynamic content
app.get('/', (req, res) => {
   // Sample marks
   const marks = 75;

   // Check marks and determine message
   let message;
   if (marks >= 80) {
       message = 'Excellent!';
   } else if (marks >= 60) {
       message = 'Good job!';
   } else {
       message = 'You can do better!';
   }

   // Render HTML with message
   res.render('index', { message });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
   console.log('Server is running on port', PORT);
});
       


<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Node.js If...Else Example</title>
</head>
<body>
   <h1>Marks Checking Result</h1>
   <p><%= message %></p>
</body>
</html>

Explanation: In the above example, we check the marks obtained by a student using if...else statements in the Node.js server. Based on the marks, a corresponding message is determined and passed to the HTML file (index.html) using a templating engine. The HTML file then displays the message to the user.

While Loop in Node.js

In Node.js, you can use a while loop to execute a block of code repeatedly as long as a specified condition is true. Let's see an example of how to use a while loop in Node.js and display the result in HTML.


Example: Counting Down

Suppose we have a Node.js server that counts down from a specified number and displays each count in HTML.


Code Example

// server.js
const express = require('express');
const path = require('path');
const app = express();

// Set up view engine and views directory
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Route to serve HTML file with dynamic content
app.get('/', (req, res) => {
   // Initialize countdown
   let count = 5;

   // Countdown loop
   let countdown = '';
   while (count > 0) {
       countdown += count + '... ';
       count--;
   }

   // Render HTML with countdown
   res.render('index', { countdown });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
   console.log('Server is running on port', PORT);
});
       


<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Node.js While Loop Example</title>
</head>
<body>
   <h1>Countdown</h1>
   <p><%= countdown %></p>
</body>
</html>

Explanation: In the above example, we use a while loop in the Node.js server to count down from 5 to 1. Each count is concatenated to a string, which is then passed to the HTML file (index.html) using a templating engine. The HTML file displays the countdown to the user.

Functions and Scope in Node.js

Functions in Node.js allow you to encapsulate reusable blocks of code. Additionally, Node.js follows JavaScript's scoping rules, which determine the accessibility of variables within different parts of your code. Let's explore functions and scope in Node.js and demonstrate their usage.


Functions

Functions in Node.js can be defined using the function keyword or arrow functions () => {}. They can accept parameters and return values as needed.


Scope

Scope in Node.js refers to the accessibility of variables within different parts of your code. Variables declared inside a function have local scope, meaning they can only be accessed within that function. Variables declared outside of any function have global scope, meaning they can be accessed from anywhere in your code.


Code Example

// server.js
const express = require('express');
const path = require('path');
const app = express();

// Set up view engine and views directory
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

// Function to calculate area of a circle
function calculateArea(radius) {
   return Math.PI * radius ** 2;
}

// Route to serve HTML file with dynamic content
app.get('/', (req, res) => {
   // Call function and pass data to HTML
   const radius = 5;
   const area = calculateArea(radius);

   // Render HTML with area
   res.render('index', { area });
});

// Start the server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
   console.log('Server is running on port', PORT);
});
       


<!DOCTYPE html>
<html lang="en">
<head>
   <meta charset="UTF-8">
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   <title>Node.js Functions and Scope</title>
</head>
<body>
   <h1>Area of Circle</h1>
   <p>Area: <%= area.toFixed(2) %></p>
</body>
</html>

Explanation: In the above example, we define a function calculateArea in the Node.js server to calculate the area of a circle. We then call this function and pass the radius value to it. The calculated area is then passed to the HTML file (index.html) using a templating engine and displayed to the user.

Arrays in Node.js are used to store multiple values in a single variable. They are a fundamental data structure and can hold elements of any data type, including numbers, strings, objects, and even other arrays. Let's explore arrays and lists in Node.js and demonstrate their usage.


Arrays

Strings and String Manipulation in Node.js

Strings in Node.js are sequences of characters enclosed within single or double quotes. String manipulation involves operations such as concatenation, splitting, slicing, and searching within strings. Let's explore strings and demonstrate string manipulation in Node.js.


Strings

Strings in Node.js can be declared using single or double quotes:


const str1 = 'Hello, world!';
const str2 = "Node.js is awesome!";

String Manipulation

String manipulation involves various operations:

  • Concatenation: Combining strings together.
  • Splitting: Breaking a string into an array of substrings.
  • Slicing: Extracting a portion of a string.
  • Searching: Finding substrings within a string.
Code Example

// String Manipulation
const str1 = 'Hello';
const str2 = 'world';
const greeting = str1 + ', ' + str2 + '!'; // Concatenation
console.log(greeting); // Output: Hello, world!

const sentence = 'Welcome to Node.js!';
const words = sentence.split(' '); // Splitting
console.log(words); // Output: ['Welcome', 'to', 'Node.js!']

const subString = sentence.slice(8, 13); // Slicing
console.log(subString); // Output: 'to No'

const searchTerm = 'Node';
const found = sentence.includes(searchTerm); // Searching
console.log(found); // Output: true

Explanation: In the above example, we demonstrate string manipulation operations such as concatenation, splitting, slicing, and searching in Node.js. These operations help manipulate strings and extract useful information from them.

Type Casting and Coercion in Node.js

Type casting and coercion in Node.js involve converting data from one type to another. Type casting refers to explicitly changing the data type, while coercion refers to implicit type conversion done by JavaScript. Let's explore type casting and coercion in Node.js with examples.


Type Casting

Type casting involves explicitly converting data from one type to another:


const num1 = '10';
const num2 = Number(num1); // Explicitly cast to number
console.log(num2); // Output: 10

const str1 = 5;
const str2 = String(str1); // Explicitly cast to string
console.log(str2); // Output: '5'
Coercion

Coercion involves implicit type conversion done by JavaScript:


const num1 = '10';
const num2 = num1 * 1; // Implicit coercion to number
console.log(num2); // Output: 10

const str1 = 5;
const str2 = str1 + ''; // Implicit coercion to string
console.log(str2); // Output: '5'

Explanation: In the above examples, we demonstrate type casting and coercion in Node.js. Type casting involves explicitly converting data using functions like Number() and String(), while coercion involves implicit conversion done by JavaScript during operations like multiplication and concatenation.
